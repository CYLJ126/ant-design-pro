我有很多配置页面，我需要对其进行新增、编辑、查看详情。其中大部分配置实体的属性都比较类似，比如字符串或数字输入框、下拉选择、时间范围选择等。我希望实现一个组件，支持我传入一组配置，即可生成表单，然后支持提交保存。

样式要求：
1. 支持大、中、小三种尺寸；大尺寸高32px宽320px字体20px，中尺寸高28px宽280px字体16px，小尺寸高24px宽240px字体13px；
2. 支持选择2列或3列，从左到右，从上到下依次排列；响应式布局，自适应屏幕，自动换行；可以设置某个控件占一整行，如地址框；
3. label最长长度要限制，文字太多时，换行；
4. 每个控件框宽和高保持一致；

各控件属性名：
1. fieldName：字段名，对应后端参数；
2. fieldType：控件类型，支持：
   1. Input输入框：支持配置文本框、数字框、密码框等；
   2. Select下拉框、Checkbox.Group复选框组、TreeSelect树形选择、Cascader级联选择：支持配置单选、多选，下拉列表默认范围，或给个链接在点击时从后端获取；
   3. Switch或 Radio开关/布尔选择：是否启用、是否删除；
   4. DatePicker时间选择、DatePicker.RangePicker时间范围选择：支持配置格式化类型，如"YYYY-MM-DD"，支持精度，如年季月日还是时分秒或毫秒；
   5. 数字范围输入：金额区间、数量区间；
   6. slot/ render：自定义插槽，可自行添加不支持的控件；
   7. 其他选择器：滑块（Slider）用于范围选择、颜色选择器、评分（Rate）、上传组件、地区选择（省市区）；
3. label：显示的标签名；
4. required：是否必填；
5. canEdit：是否可编辑，默认 true，控制有些属性只可在新增时设值，不允许后续修改；
6. transformFunction：在向后端提交时的内容转换函数，默认没有，如设置则在提交前对该字段做处理，如自定义时间格式化方法；
7. placeholder：提示内容；
8. defaultValue：默认值，如时间可取当前时间；
9. options：下拉选项，用于 select 等组件；
10. optionUrl：下拉选项加载 Url；
11. optionKeyMap：自定义 options 的 label/value 字段名；
12. clearable：是否允许清空（适用于 select/date 等）；
13. dependOn：依赖条件，如我有merchant和orgCode两个框，orgCode的下拉列表向后端请求时的参数为merchant的值{”merchants": ["merchant1", "merchant2"]}；
14. visibleFunction：显隐计算函数，如组件是否显隐依赖另一控件值，计算是否显示；
15. rules：校验规则（可选，用于提交前校验）
16. render：自定义渲染函数：`(props) => ReactNode`（用于自定义类型）；
17. debounce：防抖延迟（ms），常用于 input 搜索；
18. extraProps：透传给底层组件的额外 props（如 Select 的 `filterOption`）；
19. extraClassName：自定义类名；
20. extraStyle：自定义样式；
21. onChange：值变化回调：`(value, field) => void`，如加载另一个属性的下拉列表，或清空另一属性的值等；

要求：
1. 我传入定义的属性，为我自动渲染出对应的属性框；
2. 从我的列表页（不必为我实现）点击新增或编辑，或双击行记录，进入到此页面；
3. 新增或编辑时页面底部有提交、取消按钮：点提交时，二次弹窗确认是否保存，是则向后端发起请求，保存本次新增或修改记录，点取消则回到列表页；
4. 详情页时面底部有返回按钮，点击返回则回到列表页；
5. 按钮支持配置提交和取消按钮的快捷键；
6.详情页所有属性禁用，并有两个 Tab，一个是配置的属性信息（与新增、编辑页一致）；另一个是历史记录，有字段：createTime、createBy、updateTime、updateBy；

请按上述要求为我实现一个这样的组件，如有遗漏属性或类型，可补充或优化。
